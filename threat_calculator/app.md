# Калькулятор угроз

Приложение для моделирования и оценки угроз информационной безопасности инфраструктуры умного дома. Основано на методологии **STRIDE** и формуле риска **R = P × I**.

Источник: ВКР Гунина А.А. «Анализ и моделирование угроз для инфраструктуры умного дома и перечень рекомендаций по снижению рисков информационной безопасности» (Московский Политех, 2026).

---

## Методология

### STRIDE
Каждая угроза относится к одной из шести категорий:

| Категория | Англ. | Описание | Цвет |
|-----------|-------|----------|------|
| **S** | Spoofing | Подмена субъекта или устройства | #6366f1 |
| **T** | Tampering | Изменение данных или конфигурации | #f59e0b |
| **R** | Repudiation | Невозможность доказать действие | #8b5cf6 |
| **I** | Information Disclosure | Разглашение информации | #06b6d4 |
| **D** | Denial of Service | Отказ в обслуживании | #ef4444 |
| **E** | Elevation of Privilege | Повышение привилегий | #ec4899 |

### Формула риска

```
R = P × I
```

- **P** — вероятность (1 — низкая, 2 — средняя, 3 — высокая)
- **I** — последствия (1 — низкие, 2 — средние, 3 — высокие)
- **R** — риск (от 1 до 9)

### Уровни риска

| R | Уровень | Цвет | Действие |
|---|---|---|---|
| 1–2 | Низкий | #22c55e | Принять риск |
| 3–4 | Средний | #eab308 | Мониторить |
| 6 | Высокий | #f97316 | Требует мер |
| 9 | Критический | #ef4444 | Немедленное устранение |

### Матрица рисков 3×3

```
        Последствия (I)
        1       2       3
     +-------+-------+-------+
   1 |   1   |   2   |   3   |  низкий / средний
     +-------+-------+-------+
P  2 |   2   |   4   |   6   |  средний / высокий
     +-------+-------+-------+
   3 |   3   |   6   |   9   |  средний / критический
     +-------+-------+-------+
```

## Функции

### Активы
Устройства и компоненты умного дома:

- Контроллер / хаб
- Роутер и сетевое оборудование
- Датчики (температуры, движения, дыма)
- Камеры
- Замки / двери
- Исполнительные устройства (лампы, реле)
- Облачные сервисы
- Голосовые ассистенты

Протоколы: Wi-Fi, ZigBee, Z-Wave, Bluetooth, Ethernet, LoRaWAN, Thread, Matter

### Каталог угроз
18 предопределённых угроз из ВКР:

**Spoofing (S):**
1. Подмена ZigBee-устройства
2. Подмена мобильного приложения
3. Подмена DNS-запросов

**Tampering (T):**
4. Изменение конфигурации контроллера
5. Модификация прошивки IoT-устройства
6. Перехват и модификация трафика Wi-Fi

**Repudiation (R):**
7. Отсутствие журналирования действий пользователей
8. Отсутствие аудита доступа к облачному сервису

**Information Disclosure (I):**
9. Перехват трафика ZigBee/Z-Wave
10. Утечка данных через облачный сервис
11. Перехват Bluetooth-трафика

**Denial of Service (D):**
12. Глушение ZigBee/Z-Wave сети
13. DDoS-атака на облачный сервис
14. Исчерпание ресурсов контроллера
15. Глушение Wi-Fi сети

**Elevation of Privilege (E):**
16. Получение доступа к облачной консоли администратора
17. Эскалация привилегий через уязвимость веб-интерфейса
18. Доступ к умному замку через Bluetooth

### Оценка
Для каждой угрозы задаются:
- Вероятность (P) — 1, 2 или 3
- Последствия (I) — 1, 2 или 3
- R рассчитывается автоматически

### Фильтрация
- По категории STRIDE
- По уровню риска
- По статусу (устранено / активно)

### Отчёт
Содержит:
- Список активов
- Матрицу рисков
- Реестр всех угроз с P, I, R
- Рекомендации по каждой угрозе
- Общие рекомендации по защите умного дома

## Данные

### Актив
```
{
  id: 1,
  name: "Wi-Fi роутер",
  type: "network",
  protocols: ["Wi-Fi", "Ethernet"],
  description: "Основной роутер TP-Link"
}
```

### Угроза
```
{
  id: 1,
  name: "Глушение Wi-Fi сети",
  stride: "D",
  description: "DoS-атака на Wi-Fi роутер",
  asset_id: 1,
  asset_name: "Wi-Fi роутер",
  probability: 3,
  impact: 2,
  risk_score: 6,
  risk_level: "высокий",
  recommendation: "Использовать DFS-каналы...",
  mitigated: false
}
```

### Расчёт
```
function calculateRisk(P, I) {
  const score = P * I;
  if (score <= 2) return { score, level: "низкий", color: "#22c55e" };
  if (score <= 4) return { score, level: "средний", color: "#eab308" };
  if (score <= 6) return { score, level: "высокий", color: "#f97316" };
  return { score, level: "критический", color: "#ef4444" };
}
```

## Рекомендации по снижению рисков

- Регулярно обновлять прошивки и ПО всех устройств умного дома
- Использовать сегментацию сети (гостевая / основная / IoT VLAN)
- Отключить UPnP и ненужные сетевые сервисы на роутере
- Настроить сложные уникальные пароли для каждого устройства
- Включить журналирование событий и регулярно просматривать логи
- Использовать WPA3 для Wi-Fi, AES-шифрование для ZigBee
- Включить двухфакторную аутентификацию для облачных сервисов
- Отключить удалённый доступ к камерам при отсутствии необходимости
- Настроить резервный канал связи для критических устройств
- Изолировать IoT-устройства в отдельном VLAN без доступа к интернету
