# Калькулятор угроз — приложение для моделирования угроз ИБ

## План приложения

### Цель
Автоматизировать процесс оценки угроз информационной безопасности для инфраструктуры умного дома по методологии STRIDE с расчётом риска по формуле **R = P × I** (вероятность × последствия) и генерацией рекомендаций.

### Функциональные возможности

1. **Управление активами** — CRUD для устройств умного дома (контроллеры, датчики, камеры, замки, облачные сервисы и т.д.) с указанием протоколов связи (Wi-Fi, ZigBee, Z-Wave, Bluetooth)

2. **Каталог угроз** — предопределённые угрозы из ВКР (18 угроз по категориям STRIDE), возможность добавлять свои

3. **Калькулятор риска** — пользователь выбирает P (1–3) и I (1–3), приложение считает R = P × I (1–9) и определяет уровень:
   - 1–2: низкий (зелёный)
   - 3–4: средний (жёлтый)
   - 6: высокий (оранжевый)
   - 9: критический (красный)

4. **Матрица рисков** — интерактивная тепловая карта 3×3

5. **Фильтрация и поиск** — по категории STRIDE, уровню риска, статусу устранения

6. **Отчёт** — печатная версия с матрицей рисков, реестром угроз и рекомендациями

7. **Встроенный справочник рекомендаций** — привязанных к типу угрозы и уровню риска

### Технологический стек
- **Backend:** Python (FastAPI)
- **ORM:** SQLAlchemy + SQLite
- **Шаблоны:** Jinja2 + Bootstrap 5
- **Формула расчёта:** R = P × I (из ВКР)

## Запуск приложения

```bash
cd threat_calculator
pip install -r requirements.txt
python main.py
```

Открыть в браузере: http://127.0.0.1:8000

### Порядок работы

1. На дашборде нажать «Загрузить каталог угроз из ВКР»
2. Добавить активы (устройства умного дома)
3. В окне угроз — отредактировать P и I для каждой угрозы, привязать к активу
4. На дашборде отслеживать матрицу рисков
5. Сформировать отчёт через кнопку «Отчёт»

## Реализация

### models.py — Модели данных

**Asset** (актив):
- id, name, type (тип устройства), protocols (протоколы связи), description, created_at

**Threat** (угроза):
- id, name, stride (категория STRIDE), description, asset_id, asset_name
- probability (P, 1-3), impact (I, 1-3), risk_score (R, 1-9), risk_level
- recommendation, mitigated (устранена/нет), is_custom (из каталога или своя)

### calculator.py — Ядро

- `calculate_risk(p, i)` → RiskResult(score, level, color)
- `format_risk_matrix()` → матрица 3×3 для отображения
- `StrideCategory` — enum S/T/R/I/D/E с названиями и описаниями
- `RiskLevel` — enum (низкий, средний, высокий, критический)

Формула: **R = P × I**

| P (вероятность) | I (последствия) || R | Уровень |
|---|---|---|---|
| 1 — низкая | 1 — низкие || 1-2 | Низкий |
| 2 — средняя | 2 — средние || 3-4 | Средний |
| 3 — высокая | 3 — высокие || 6 | Высокий |
| | || 9 | Критический |

### catalog.py — Справочник из ВКР

18 предопределённых угроз, сгруппированных по STRIDE:
- **S (Spoofing):** подмена ZigBee-устройства, подмена мобильного приложения, подмена DNS
- **T (Tampering):** изменение конфигурации контроллера, модификация прошивки, перехват Wi-Fi
- **R (Repudiation):** отсутствие журналирования, отсутствие аудита облака
- **I (Information Disclosure):** перехват ZigBee/Z-Wave, утечка через облако, перехват Bluetooth
- **D (DoS):** глушение ZigBee/Z-Wave, DDoS облака, исчерпание ресурсов, глушение Wi-Fi
- **E (Elevation of Privilege):** доступ к облачной консоли, эскалация через веб-интерфейс, доступ к замку через BLE

### main.py — REST API и страницы

- `GET /` — дашборд со статистикой и матрицей рисков
- `GET/POST /assets`, `/assets/new`, `/assets/{id}/edit`, `/assets/{id}/delete` — управление активами
- `GET/POST /threats`, `/threats/new`, `/threats/{id}/edit`, `/threats/{id}/delete`, `/threats/{id}/toggle` — управление угрозами
- `GET /report` — отчёт для печати
- `POST /threats/load-catalog` — загрузка каталога угроз из ВКР
